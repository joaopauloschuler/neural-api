# StarCoder2Complete: CPU code completion with an imported StarCoder2 model

This example is a minimal **code-completion** harness for the CAI
**StarCoder2** importer (`BuildStarCoder2FromSafeTensors` /
`BuildFromPretrained` model_type `starcoder2`, in
`neural/neuralpretrained.pas`). StarCoder2 (bigcode/`starcoder2-3b` /
`starcoder2-7b` / `starcoder2-15b`) is the first **code-specialised**
decoder in the repo. Point the demo at a model directory and its stock BPE
`tokenizer.json`, give it a code prompt, and it greedily extends the prompt
one argmax token at a time and prints the decoded continuation.

```
StarCoder2Complete <model-dir-or-safetensors> <tokenizer.json> \
                   [SeqLen] [MaxNewTokens] [prompt...]
```

`<model-dir-or-safetensors>` is a directory holding `config.json` +
`model.safetensors` (or `pytorch_model.bin`), or the weights file directly
(then `config.json` is read from its directory). `SeqLen` defaults to 256 —
keep it small on the real 3B checkpoint, the full 16384-token context is
slow and memory-hungry on CPU. `MaxNewTokens` defaults to 48. Remaining
arguments are joined with spaces as the prompt (default: a short Python
function header). Example after downloading `bigcode/starcoder2-3b` to
`./starcoder2-3b`:

```
StarCoder2Complete ./starcoder2-3b ./starcoder2-3b/tokenizer.json 256 64 \
  'def fibonacci(n):'
```

## What makes StarCoder2 architecturally distinct

It is NOT on the Llama (RMSNorm + SwiGLU) path. It pairs the modern
attention front-end — **RoPE** on q/k, **GQA** (`num_key_value_heads`), an
optional **sliding window** applied to every layer when `sliding_window` is
set — with three GPT-2-flavoured pieces the RMSNorm/SwiGLU importers never
exercise (handled for you by the importer; see the STARCODER2 IMPORT section
of the `neuralpretrained.pas` unit header):

* **Biased `nn.LayerNorm` norms** (NOT RMSNorm): `input_layernorm`,
  `post_attention_layernorm` and the final `model.norm` are
  `TNNetTokenLayerNorm` with a learned gain **and bias**, `eps =
  norm_epsilon`.
* **bias=True (`use_bias`) on every linear** — q/k/v AND `o_proj` (the exact
  path OLMo-2 deliberately *rejects*) plus both MLP matrices. `lm_head` is
  always bias-free.
* **A plain two-matrix `gelu_pytorch_tanh` FFN** — `c_fc -> GELU -> c_proj`,
  with NO SwiGLU gate (`TNNetGELU` is HF's tanh GELU).

q/k rows carry the same per-head rotate_half permutation as the Llama path;
v/o load straight. Embeddings are usually untied (a separate `lm_head`).

## Parity

The committed pico fixtures `tests/fixtures/tiny_starcoder2{,_window}.*`
(generated by `tools/starcoder2_tiny_fixture.py`, no download — a random
pico config) pin both a **full-attention** and a **sliding-window** config
to the HuggingFace `transformers` float64 oracle within `1e-4`
(`TestStarCoder2LogitParity` / `TestStarCoder2WindowLogitParity` in
`tests/TestNeuralPretrained.pas`). The fixture generator also asserts the
LayerNorm biases, the `o_proj` bias and the sliding window each move the
logits, so the importer's bias/window paths are genuinely covered.

This demo always builds the net `pInferenceOnly` (training volumes freed
during construction) and runs a full forward per token (no KV cache).
