# Summarize — abstractive summarization with a pretrained BART

This example performs **abstractive text summarization** on CPU using a
pretrained HuggingFace **BART** checkpoint imported by
`BuildBartFromSafeTensors` (neural/neuralpretrained.pas). BART is the
dominant pretrained encoder-decoder for summarization: a bidirectional
encoder reads the article and a causal decoder with cross-attention writes
the summary.

Supported checkpoints are the standard `model_type "bart"` summarization
models, e.g.:

- [`sshleifer/distilbart-cnn-12-6`](https://huggingface.co/sshleifer/distilbart-cnn-12-6) (fastest published CNN model)
- [`facebook/bart-large-cnn`](https://huggingface.co/facebook/bart-large-cnn) (heavier)

## How it works

1. The checkpoint's GPT-2 byte-level BPE `tokenizer.json` encodes the
   article (no SentencePiece needed — `TNeuralHFTokenizer` reads it
   directly), framed as `bos … eos`.
2. `BuildBartFromSafeTensors` builds the encoder + decoder net pair
   (learned `+2`-offset absolute positions, `layernorm_embedding`,
   post-norm blocks, exact-erf GELU FFN, tied LM head + `final_logits_bias`).
3. `DecodeSeq2SeqBeamSearch` (neural/neuraldecode.pas) runs token-id beam
   search through the two-net pair (`decoder_start_token_id = eos`, the BART
   shift), terminating on `eos`.
4. The summary is decoded back to text and — when a reference summary is
   supplied — scored with ROUGE-1 / ROUGE-2 / ROUGE-L F1
   (neural/neuralnlpmetrics.pas).

## Build

From the repository root (always cap memory):

```bash
LAZUTILS_PATH=$(find /usr/lib/lazarus /usr/share/lazarus -name utf8process.ppu -printf '%h\n' | head -1)
ulimit -v 3000000
fpc -O2 -Mobjfpc -Sh -Funeural -Fu"$LAZUTILS_PATH" \
    -FUexamples/Summarize examples/Summarize/Summarize.lpr
```

(`LAZUTILS_PATH` is needed so the tokenizer's non-ASCII JSON keys decode
correctly — see the `hf-tokenizer-and-fpjson-traps` note.) Or open
`Summarize.lpi` in Lazarus.

## Run

Point it at a directory containing `model.safetensors`, `config.json` and
`tokenizer.json`:

```bash
ulimit -v 3000000
./examples/Summarize/Summarize /path/to/distilbart-cnn-12-6 \
    -af article.txt -rf reference.txt -enc 256 -dec 64 -beam 4
```

With no `-af`/`-ref` it summarizes a built-in demo article and scores
against its reference. Options:

| Option | Meaning |
|--------|---------|
| `-a "text"` / `-af file` | source article (text / file) |
| `-ref "text"` / `-rf file` | reference summary for ROUGE |
| `-enc N` | encoder window (default 256; article truncated to fit) |
| `-dec N` | max summary length (default 64) |
| `-beam N` | beam width (default 4; 1 = greedy) |
| `-lp f` | length-penalty alpha (default 1.0, Wu et al. 2016) |

**CPU note.** `distilbart-cnn-12-6` is the fastest published CNN model;
`bart-large-cnn` is much heavier. Keep `-enc`/`-dec`/`-beam` small for a
quick CPU demo (the decoder re-forwards once per live beam per step over the
full 50k-token vocabulary, so cost scales with `beam × dec × vocab`).

## Verification

The importer is parity-checked against a pico float64 HF oracle in
`tests/TestNeuralPretrained.pas` (`TestBartParity`, fixture generated by
`tools/bart_tiny_fixture.py`): max |diff| ~1.5e-6 (encoder hidden states)
and ~3.6e-6 (decoder logits) against the 1e-4 gate.
