# VQModelImport — VQGAN / discrete image-tokenizer import

Imports a pretrained **diffusers VQModel** (VQGAN) into a CAI `TNNetVqModel`
via `BuildVqModelFromSafeTensors` (`neural/neuralpretrained.pas`) and
round-trips a tiny image through the codebook:

```
image  ->  encoder + quant_conv  ->  nearest-neighbour codebook lookup
       ->  grid of integer token IDs
       ->  gather codebook embeddings  ->  post_quant_conv + decoder  ->  image
```

This is the discrete tokenizer used by autoregressive / masked image
generators (MaskGIT, Parti, LlamaGen) to turn an image into a sequence of
codebook token IDs a stock Llama/GPT LM can model, and back again.

It reuses the landed AutoencoderKL VAE encoder/decoder ResNet+attention blocks
(`BuildVaeEncoder` / `BuildVaeDecoder`); the genuinely new pieces are the
nearest-neighbour codebook lookup (`EncodeImageToTokens`, argmin squared-L2 to
`quantize.embedding.weight`) and the inverse gather (`DecodeTokensToImage`),
both plain Pascal on the holder class `TNNetVqModel` (no new `TNNet*` layer).
Unlike AutoencoderKL there is **no** 0.18215 latent scaling.

## Run

```
fpc -Fu../../neural -Mobjfpc -Sh -O2 VQModelImport.lpr
./VQModelImport                              # committed pico fixture
./VQModelImport model.safetensors config.json  # real diffusers VQModel
```

With no arguments it loads the committed pico fixture
(`tests/fixtures/tiny_vqmodel.*`), the same one the parity tests use
(`TestVqModelEncodeParity` — exact integer token-ID match vs a numpy float64
oracle; `TestVqModelDecodeParity` / `TestVqModelRoundTrip` — decoded pixels
`< 1e-4`). A random pico VQModel does not reconstruct well; a real pretrained
checkpoint does. The point is the discrete image → tokens → image pipeline
runs end to end on CPU.

The pico fixture is generated by `tools/vqmodel_tiny_fixture.py` (a
self-contained numpy float64 oracle; diffusers is not required).
